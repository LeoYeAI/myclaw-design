## Галерея демо

### Советник по направлению дизайна

Когда требования размыты, навык выбирает 3 дифференцированных направления из 5 школ × 20 дизайн-философий, генерирует параллельные демо и даёт вам выбрать.


### Прототип iOS-приложения

Точная рамка iPhone 15 Pro (Dynamic Island / строка состояния / Home Indicator) · Многоэкранная навигация на основе состояний · Реальные изображения с Wikimedia/Met/Unsplash · Автоматическое тестирование кликов Playwright перед сдачей.


### Движок моушн-дизайна

Модель таймлайна Stage + Sprite · `useTime` / `useSprite` / `interpolate` / `Easing` — четыре API покрывают все потребности анимации. Одна команда экспортирует MP4 / GIF / 60fps с интерполяцией / финальный монтаж с BGM + SFX.


### Презентации + Редактируемый PPTX

HTML-first слайды с автомасштабированием, навигацией с клавиатуры и заметками докладчика. Экспорт в редактируемый PPTX с нативными текстовыми полями — не скриншоты, вставленные на слайды.


### Live Tweaks

Настройка параметров в реальном времени — меняйте цветовые схемы, макеты, типографику и плотность без перегенерации. Изменения сохраняются через localStorage.


### Инфографика

Макеты на основе данных, готовые к печати, с точной типографикой. Экспорт в PDF/PNG/SVG.


### Экспертная оценка по 5 измерениям

Философская согласованность / Визуальная иерархия / Проработка деталей / Функциональность / Инновационность — каждый критерий оценивается от 0 до 10 с радарной диаграммой, списком «Сохранить», списком «Исправить» (ранжированным по серьёзности) и «Быстрыми победами».

## Как это работает

### Протокол брендовых ассетов

Навык не угадывает ваш бренд. Он следует строгому 5-шаговому протоколу:

1. **Запросить** — Запрашивает логотип, фото продукта, скриншоты UI, цветовую палитру, типографику
2. **Найти** — Сканирует официальные сайты, пресс-киты, магазины приложений в поисках ассетов
3. **Скачать** — Загружает реальные файлы (SVG логотипа, hero-изображения продукта, скриншоты UI)
4. **Проверить** — Контролирует разрешение, прозрачность, актуальность версии
5. **Зафиксировать** — Записывает `brand-spec.md` со всеми путями к ассетам; CSS-переменные обеспечивают единообразие


> **Почему это важно:** Без реальных брендовых ассетов каждый ИИ-сгенерированный дизайн выглядит одинаково — типовые градиенты, иконки-заглушки, нулевая узнаваемость бренда. Протокол стоит 30 минут на старте, но экономит 1–2 часа доработок.

### Рабочий процесс младшего дизайнера

Навык работает как младший дизайнер, отчитывающийся перед вами:

1. **Сначала показать допущения** — Записывает рассуждения + заглушки до написания кода
2. **Получить одобрение** — Ждёт вашего направления перед заполнением деталей
3. **Итерировать** — Показывает прогресс по ходу работы, а не только финальный результат
4. **Верифицировать** — Запускает скриншоты Playwright + проверку ошибок консоли перед сдачей


### Анти-ИИ-шлак

Каждое дизайн-решение проверяется по строгому анти-шлак списку:

| Избегать | Использовать вместо |
|---|---|
| Фиолетовые градиенты | Цвета бренда / гармоники `oklch()` |
| Эмодзи вместо иконок | Честные заглушки или реальные ассеты |
| Скруглённые карточки + акцент левой границы | Чистые границы, заслуженные контентом |
| SVG-нарисованные лица/объекты | Реальные изображения или честные заглушки |
| CSS-силуэты вместо фото продукта | Реальные фото продукта из протокола бренда |
| Inter/Roboto/системные шрифты как дисплейные | Выразительная пара дисплейного + текстового шрифта |

## Стартовые компоненты

Готовые компоненты для немедленного использования:

| Компонент | Применение |
|---|---|
| `assets/ios_frame.jsx` | Рамка iPhone 15 Pro с Dynamic Island, строкой состояния, Home Indicator |
| `assets/android_frame.jsx` | Рамка Android-устройства |
| `assets/macos_window.jsx` | Окно macOS с кнопками-светофорами |
| `assets/browser_window.jsx` | Окно браузера с адресной строкой + вкладками |
| `assets/animations.jsx` | Движок Stage + Sprite + useTime + Easing |
| `assets/deck_index.html` | Сборщик многофайловых презентаций |
| `assets/deck_stage.js` | Web-компонент однофайловой презентации |
| `assets/design_canvas.jsx` | Сетка сравнения вариантов бок о бок |

## Аудио-ассеты

6 BGM-треков, подобранных под сцены + 37 категоризированных SFX-файлов для анимации продакшн-качества:

- **BGM**: техно / рекламный / образовательный / туториал (+ альтернативные варианты)
- **SFX**: клавиатура, терминал, переход, удар, магия, обратная связь, UI, контейнер, прогресс

## Структура проекта

```
myclaw-design/
├── SKILL.md              # Основные инструкции навыка (загружаются OpenClaw)
├── assets/               # Стартовые компоненты, BGM, SFX, витрины
│   ├── *.jsx             # React-компоненты (рамки iOS/Android/macOS и др.)
│   ├── bgm-*.mp3         # 6 фоновых музыкальных треков под сцены
│   ├── sfx/              # 37 категоризированных звуковых эффектов
│   └── showcases/        # 24 готовых дизайн-образца (8 сцен × 3 стиля)
├── references/           # Углублённые руководства (загружаются по запросу)
│   ├── animation-*.md    # Лучшие практики анимации + подводные камни
│   ├── design-styles.md  # База данных 20 дизайн-философий
│   ├── react-setup.md    # Техническая настройка React + Babel
│   ├── slide-decks.md    # Руководство по архитектуре слайдов
│   ├── video-export.md   # Пайплайн экспорта MP4/GIF
│   └── ...               # 18 справочных файлов всего
└── scripts/              # Скрипты автоматизации
    ├── render-video.js   # HTML → MP4 (25fps)
    ├── convert-formats.sh # Интерполяция 60fps + GIF
    ├── add-music.sh      # Сведение BGM + SFX
    ├── export_deck_*.mjs # Экспорт PDF + PPTX
    └── verify.py         # Верификация Playwright
```

## Требования

- [OpenClaw](https://github.com/openclaw/openclaw) (любая актуальная версия)

- [Playwright](https://playwright.dev/) (для верификации + экспорта видео)
- ffmpeg (для конвертации видеоформатов + сведения аудио)
